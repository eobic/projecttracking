About the Data Pillars

Data Intake
Bringing new and updated data into Snowflake from source systems. Includes onboarding acquisitions, adding vendor fields, uploading historical data, and fixing source mapping. If the work makes data available in Snowflake for the first time, it belongs here.
Data Modeling
Transforming raw data into structured, business-ready datasets. Includes building fact/dimension tables, churn/WIP models, KPI logic, and mapping tables. If the work changes business logic or how metrics are calculated, it belongs here.
Reporting
Delivering insights through dashboards and analysis. Includes building Power BI reports, DAX measures, ad hoc data requests, filters, and report-level bugs. If the work happens in Power BI or focuses on how users consume data, it belongs here.
Governance & Architecture
Improving the structure, reliability, and scalability of the data platform. Includes orchestration tooling, data ticketing, RLS/security, schema restructuring, and preventing refresh failures. If the work strengthens the platform itself rather than a single report, it belongs here.
AI Pipeline
Building automation and intelligent workflows using AI. Includes LLM integrations, automated summarization, candidate review, compliance automation, and AI-driven internal tools. If the project leverages AI to automate or enhance decision-making, it belongs here.

Project Difficulty Tiers

Tier 1 Tactical
Small, contained tasks with minimal impact. Examples: minor visual updates, simple DAX fixes, adding a column, basic ad hoc data pulls.
Tier 2 Moderate
Clear scope but limited cross-system impact. Examples: adding a new KPI, modifying a dbt model, onboarding a small data source, fixing a defined data mismatch.
Tier 3 Complex
Cross-functional or multi-layer impact (Snowflake → dbt → Power BI). Examples: updating business logic affecting executive dashboards, onboarding a new acquisition, investigating root-cause metric discrepancies.
Tier 4 Strategic
Architectural or firm-impact work with high visibility. Examples: restructuring core fact tables, implementing orchestration tooling, redesigning KPI frameworks.
Tier 5 Transformational
Firm-wide or future-facing initiatives that change how the organization operates. Examples: enterprise AI implementations, platform redesigns, data governance overhauls.
